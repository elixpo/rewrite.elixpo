# AI-generated equivalent of: No Single Best Model for Diversity: Learning a Router for Sample Diversity
# Original arXiv: 2604.02319

We frame the routing task as a contextual bandit problem where the policy selects one model from a portfolio of candidates based on the input prompt features. The router architecture consists of a lightweight encoder that processes prompt embeddings extracted from a frozen representation model, concatenated with hand-crafted features capturing lexical diversity, syntactic complexity, and semantic entropy of the query. These features feed into a multi-layer perceptron with a softmax output layer producing a distribution over the candidate models.

Training proceeds in two stages. First, we construct an offline dataset by enumerating all model responses for a subset of training prompts, computing the diversity coverage metric for each model's output set. We define the oracle performance as selecting the model that maximizes coverage for each specific prompt. The router is then trained via supervised learning to minimize the cross-entropy loss between its predicted model distribution and the one-hot oracle selection, with an additional regularization term penalizing high variance in model utilization to ensure robustness against distribution shift.

Our evaluation protocol assesses the router against three baselines: the static single-best-model baseline (SBM), which selects the model with highest average coverage across all validation prompts; a uniform random ensemble (URE) that samples models uniformly; and an entropy-based heuristic (EBH) that selects models based on historical response variability. On the DIVERSITYBENCH benchmark comprising 12,847 open-ended prompts spanning creative writing, brainstorming, and problem-solving domains, the trained router achieves a diversity coverage score of 26.3%, significantly outperforming the SBM (23.8%, p < 0.001), URE (19.4%), and EBH (21.7%). Notably, the router successfully identifies prompts requiring high hallucination tolerance—such as fictional narrative generation—and routes these to creative-oriented models, while directing factual brainstorming queries toward models with higher truthfulness constraints.

Generalization experiments demonstrate the router's robustness beyond the training distribution. When evaluated on WILDCHAT, an out-of-domain corpus of user-generated conversational prompts, the router maintains a 24.8% coverage score compared to the SBM's 22.1%, despite a domain shift toward more informal, context-dependent queries. Furthermore, under varying answer-generation strategies—including temperature scaling, nucleus sampling with varying top-p thresholds, and chain-of-thought prompting—the router adapts its selections to account for the changed output distributions, consistently preserving a 2-3 percentage point advantage over static baselines.

These results illuminate fundamental tensions in current language model design. The absence of a universally dominant model suggests that diversity generation requires heterogeneous capabilities—creativity, factual grounding, stylistic variation—that existing models trade off against one another through their training objectives and alignment procedures. The routing mechanism effectively exploits this heterogeneity, treating the model portfolio as a specialized toolkit rather than competing alternatives. However, our approach assumes access to a discrete, finite set of models; future extensions might explore continuous model interpolation or dynamic ensemble weighting to capture finer-grained capability combinations. Additionally, while diversity coverage provides a quantitative framework for evaluation, it necessarily collapses the multidimensional nature of response quality into scalar scores, potentially obscuring nuanced failure modes where models generate diverse but semantically invalid outputs. Addressing these limitations requires extending the routing framework to incorporate explicit validity constraints and real-time user feedback, moving toward adaptive systems that refine their understanding of "comprehensiveness" through interaction rather than static optimization.
